## 高效刷题指南

### 时间规划

- **每道 Easy**：10–15 分钟（含想 + 写 + 对拍）
- **每道 Medium**：15–25 分钟；卡住超 20 分钟就先看提示
- **每道 Hard**：25–40 分钟；可分段做（先暴力再优化）
- **每天 5 题**：建议 1–1.5 小时专注块，早/晚各一块更易坚持

### 做题前 30 秒自问

1. 题意和示例都清楚了吗？边界（空、单元素、重复）考虑了吗？
2. 能一句话说出「用什么套路」吗？（双指针 / 滑动窗口 / DFS / DP...）
3. 目标时间/空间复杂度大概是多少？

### 复习节奏（艾宾浩斯简化版）

- **第 1 天**：做 Day N
- **第 2 天**：做 Day N+1 前，花 5 分钟快速过一遍 Day N 的「当天重点」
- **第 4 天**：做 Day N+3 前，闭眼回忆 Day N 的 2–3 道核心题的思路
- **第 8 天**：做 Day N+7 前，挑 Day N 里最卡的一道重写一遍

### 每日完成后自检

- 今天 5 题的「核心思路」能脱口而出吗？
- 有 1 道以上完全没想出来、直接看答案的吗？若有，标记「需二刷」
- 明天的技能跟今天有关吗？有的话提前扫一眼模板

### 核心模板速查（卡住时看一眼）


| 套路       | 一句话                                        | 关键代码                                                |
| -------- | ------------------------------------------ | --------------------------------------------------- |
| 快慢指针     | slow 写、fast 扫，满足条件才写                       | `if cond: arr[slow]=arr[fast]; slow+=1`             |
| 对撞指针     | l,r 两端向中间，根据比较移动                           | `while l<r: if arr[l]+arr[r]...</`                  |
| 滑动窗口     | r 扩、不满足时 l 收                               | `while valid: update; r++; while !valid: l++`       |
| 区间合并     | 按起点排序，重叠就扩展右端点，否则结算                     | `if cur[1]>=nxt[0]: cur[1]=max(...) else append`    |
| 单调栈      | 维护递减/递增栈，出栈时计算宽度/面积/水量                 | `while stk and stk[-1]<x: stk.pop(); stk.append(x)` |
| 二分       | 找满足条件的边界，注意 l,r 的取等                        | `while l<=r: m=(l+r)//2; if cond: r=m-1 else l=m+1` |
| 链表虚拟头    | dummy 挂在 head 前，返回 dummy.next               | `dummy=ListNode(0); dummy.next=head`                |
| BFS 层序遍历 | queue，每层 for _ in range(len(q))            | `while q: for _ in range(len(q)): ...`              |
| DFS 递归   | 选/不选 or 枚举下一个，记得恢复                         | `path.append(x); dfs(); path.pop()`                 |
| 拓扑排序     | 入度为 0 入队，出队时减邻居入度，全出队则无环              | `if indeg[v]==0: q.append(v); cnt+=1`               |
| Trie     | 逐字符建子节点，is_end 标记单词结尾                     | `node=node.children.setdefault(c, {})`              |
| 回溯       | 选择→递归→撤销；组合用 start 去重，排列用 visited        | `path.append(x); dfs(start+1); path.pop()`          |
| Kadane   | 以 i 结尾的最大和 = max(nums[i], dp[i-1]+nums[i]) | `cur=max(x, cur+x); best=max(best,cur)`             |

| 2D DP    | dp[i][j] 由上/左/左上转移，注意初始化首行首列             | `dp[i][j]=f(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])` |
| LRU 设计   | 哈希表 + 双向链表，get/put 移到头，满时删尾               | `dict[key]=node; move_to_head(node)`                |
| 对顶堆      | 大顶堆存小半、小顶堆存大半，维护 size 差 ≤ 1              | `heappush(lo,-x); heappush(hi,x); rebalance()`     |

## Day 1 — day01-two-pointers（Two Pointers 基础）

**核心能力**：快慢指针、对撞指针、原地修改数组。  
**当天重点**：88/27/26 是「快慢指针」三板斧（归并、删除、去重）；125/392 是对撞指针和双指针遍历，建立左右指针同时移动的直觉。  
**拓展**：977 Squares of Sorted Array（有序数组平方）、986 Interval List Intersections（区间列表交集）

**技能标签**：双指针、原地数组操作、字符串对撞指针。  
**选题逻辑**：先用 88/27/26 在「有序数组 + 原地修改」场景下熟悉快慢指针的基本写法；再通过 125/392 把指针从数组延伸到字符串和两个序列的同步遍历，形成「一前一后」「两端向中间」「一个追一个走」这几种最基础直觉。  
**思维套路**：  
- 快慢指针写入：`fast` 负责扫描原数组，`slow` 只在「保留当前元素」时写入并前进，最终 `slow` 给出有效长度。  
- 对撞指针：左右指针从两端向中间逼近，根据当前左右元素是否满足条件决定移动哪一侧。  
**题目对应能力**：88 逆向双指针归并避免覆盖；27 快慢指针删除元素，体会「只保留想要的」；26 覆盖多余元素实现去重；125 字符串对撞指针+字符过滤；392 两个序列的子序列匹配直觉。  
**常见坑点**：快慢指针返回长度 off-by-one；88 中正向写入导致覆盖；对撞指针时忽略非字母数字或大小写转换。  
**自测问题**：不看代码能写出快慢指针去重模板吗？遇到「删除/保留数组元素」「判断回文/子序列」能立刻联想到今天哪道题？

## Day 2 — day02-two-pointers（Two Pointers 进阶）

**核心能力**：对撞指针进阶、排序后双指针、盛水/接雨类问题。  
**当天重点**：80 在 26 基础上加「最多保留 2 个」的计数；167/11/15 是「有序数组 + 对撞」的经典组合；42 接雨水既可双指针也可单调栈，建议两种写法都过一遍。  
**拓展**：18 4Sum（四数之和）、75 Sort Colors（荷兰国旗/三路 partition）

**技能标签**：对撞指针进阶、排序+双指针去重、盛水/接雨问题。  
**选题逻辑**：80 在 Day 1 的 26 基础上通用化为「最多保留 k 个」；167/11 巩固对撞指针在求和、面积场景的应用；15 把对撞指针嵌入排序+遍历框架，是面试超高频题；42 接雨水是双指针和单调栈的综合压轴。

## Day 3 — day03-sliding-window（Sliding Window）

**核心能力**：固定/可变窗口、左右指针收缩、字符计数。  
**当天重点**：219 是简单窗口入门；209 是最小可变窗口模板；3 是无重复子串的经典题；76 是最小覆盖子串，需要 needs/window 双计数 + 收缩条件判断，是滑动窗口 Hard 标杆。  
**拓展**：424 Longest Repeating Character Replacement（最多替换 k 次）、992 Subarrays with K Different Integers（恰含 k 个不同整数）

**技能标签**：固定窗口、可变窗口（最大/最小）、字符计数窗口。  
**选题逻辑**：219 用固定大小窗口+哈希集合做入门；209 练习「满足条件就收缩左边界」的最小可变窗口模板；3 用集合维护无重复字符的最大窗口；76 是滑动窗口的 Hard 标杆，needs/window 双计数+valid 收缩判断，综合前面所有技巧。  
**思维套路**：  
- 可变窗口模板：右指针扩张，当窗口满足条件时收缩左指针更新答案（求最小）；或当窗口不满足条件时收缩左指针（求最大）。  
- 字符计数窗口：用 needs 记录目标需求，window 记录当前窗口内计数，valid 计数有多少字符已满足。  
**题目对应能力**：219 固定窗口+集合判重；209 最基础的「和 ≥ target」可变窗口；3 集合/哈希维护窗口内无重复；76 双哈希计数+收缩条件的完整实现。  
**常见坑点**：209 中忘记在收缩时更新答案；3 中用数组下标而非集合导致逻辑复杂；76 中 valid 的增减时机搞错（只在恰好满足/恰好不满足时变化）。  
**自测问题**：能默写「最小覆盖子串」的完整滑窗代码吗？遇到「最长/最短满足某条件的子串/子数组」能立刻判断用哪种窗口模板？

## Day 4 — day04-sliding-window-greedy（Sliding Window + Greedy）

**核心能力**：固定窗口 + 词频、股票贪心、跳跃可达性。  
**当天重点**：30 是固定窗口 + 词频哈希，边界较多；121/122/55 是经典贪心：维护最小值、累加上升段、维护可达最远位置。  
**拓展**：406 Queue Reconstruction by Height（身高重排）、763 Partition Labels（划分字母区间）

**技能标签**：固定窗口+词频哈希、一次遍历贪心、跳跃可达性。  
**选题逻辑**：30 是滑动窗口的收尾题，用固定长度窗口+词频匹配；121/122/55 开启贪心专题——维护历史最小值、累加上升段、维护可达最远位置。从窗口过渡到贪心，共同点是「一次遍历+维护状态」。  
**思维套路**：  
- 固定窗口+词频：窗口大小 = 单词长度 × 单词数量，按单词为单位滑动，用哈希表比对词频。  
- 一次遍历贪心：扫描过程中维护一个关键变量（最小值/最远位置/累计利润），每步做局部最优决策。  
**题目对应能力**：30 固定窗口按词切分+词频匹配；121 维护前缀最小价格求最大差值；122 只要明天比今天贵就买卖；55 维护 maxReach 判断能否到终点。  
**常见坑点**：30 中窗口起点需要枚举 0 到 wordLen-1 共 wordLen 个起点；121 中混淆买入/卖出时机；55 中忘记判断当前位置是否可达。  
**自测问题**：能说出 30 为什么需要多个起点吗？121/122/55 三道题的贪心策略能分别一句话概括吗？

**当天重点**：9 反转一半比回文；66 数组进位；172 统计因子 5；50 快速幂；70 爬楼梯是 DP 最入门题。  
**拓展**：263 Ugly Number（因子判断）、91 Decode Ways（简单 1D DP）

**技能标签**：数字回文判断、数组进位模拟、因子计数、快速幂、Fibonacci DP。  
**选题逻辑**：9/66/172/50 是四道数学基础题——回文数、加一、阶乘尾零、快速幂；70 爬楼梯是 DP 最入门题（本质是 Fibonacci）。数学题练基础功，70 为后面两天的 DP 做铺垫。  
**思维套路**：  
- 反转一半判回文：反转后半部分数字，和前半部分比较，避免溢出。  
- 快速幂：`x^n = (x^(n/2))^2`，n 为奇数多乘一个 x，O(log n)。  
- Fibonacci DP：`dp[i] = dp[i-1] + dp[i-2]`，只需两个变量滚动。  
**题目对应能力**：9 反转一半数字比较；66 从末位开始处理进位；172 统计 n! 中因子 5 的个数（5, 25, 125...）；50 递归或迭代快速幂；70 `f(n) = f(n-1) + f(n-2)` 滚动数组。  
**常见坑点**：9 中负数直接返回 false、末位为 0 但非零的数不是回文；66 中全是 9 的情况（如 999→1000）；50 中 n 为负数时取倒数；70 中 base case 的设定。  
**自测问题**：172 中为什么只需要数因子 5 而不数因子 2？50 快速幂的时间复杂度为什么是 O(log n)？

## Day 30 — day30-dp-1d（1D DP）

**核心能力**：打家劫舍、字典切分、完全背包、LIS、股票多状态。  
**当天重点**：198 相邻不取；139 字典切分 DP；322 完全背包最少硬币；300 LIS 可二分优化；123 股票 III 是两笔交易的状态机。  
**拓展**：213 House Robber II（环形）、279 Perfect Squares（完全背包 / BFS）

**技能标签**：打家劫舍、字典切分 DP、完全背包、最长递增子序列、股票多交易状态机。  
**选题逻辑**：198 打家劫舍是 DP 入门经典（相邻不取）；139 字典切分 DP 练习「枚举上一个切分点」的转移；322 完全背包求最少硬币；300 LIS 可 O(n²) DP 或 O(n log n) 二分优化；123 股票 III 两笔交易是多状态 DP 的 Hard 题。  
**思维套路**：  
- 打家劫舍：`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`，选或不选当前房屋。  
- 完全背包：`dp[amount] = min(dp[amount], dp[amount-coin] + 1)`，每个硬币可以用无限次。  
- LIS 二分优化：维护 tails 数组，`tails[i]` 是长度为 i+1 的递增子序列的最小末尾，用 bisect_left 更新。  
- 股票状态机：定义 buy1/sell1/buy2/sell2 四个状态，逐天转移。  
**题目对应能力**：198 选/不选的基础 DP；139 枚举字典中的词作为最后一段；322 完全背包最少硬币数；300 经典 LIS（DP 或二分）；123 四状态转移处理两笔交易。  
**常见坑点**：198 中 dp 数组初始化；139 中子串判断的效率（用 set 存字典）；322 中 `dp[0]=0` 其余初始化为 `inf`；300 二分优化中 tails 数组的含义；123 中 buy 状态初始化为 `-inf` 或 `-prices[0]`。  
**自测问题**：能写出 322 完全背包的状态转移方程吗？300 的 O(n log n) 二分优化中 tails 数组每个元素代表什么？

## Day 31 — day31-dp-2d（2D / Multi-D DP）

**核心能力**：三角形路径、网格最短路径、障碍路径、回文、交错字符串。  
**当天重点**：120 自底向上；64/63 网格 DP；5 中心扩展或二维 DP；97 交错字符串是 2D 经典。  
**拓展**：131 Palindrome Partitioning（回文划分）、312 Burst Balloons（区间 DP）

**技能标签**：三角形路径 DP、网格最短路径、障碍网格、最长回文子串、交错字符串。  
**选题逻辑**：120 三角形自底向上 DP 是二维 DP 的入门；64/63 网格最短路径和障碍网格是经典网格 DP；5 最长回文子串可用中心扩展 O(n²) 或二维 DP；97 交错字符串是二维 DP 的经典应用。  
**思维套路**：  
- 网格 DP：`dp[i][j]` 由上方 `dp[i-1][j]` 和左方 `dp[i][j-1]` 转移，注意首行首列的初始化。  
- 回文 DP/中心扩展：中心扩展从每个位置向外扩展；DP 则是 `dp[i][j] = (s[i]==s[j]) and dp[i+1][j-1]`。  
- 交错字符串 DP：`dp[i][j] = (dp[i-1][j] and s1[i-1]==s3[i+j-1]) or (dp[i][j-1] and s2[j-1]==s3[i+j-1])`。  
**题目对应能力**：120 自底向上逐行取 min 转移；64 网格从左上到右下的最小路径和；63 有障碍的网格路径计数；5 中心扩展或区间 DP 求最长回文；97 二维 DP 判断交错可行性。  
**常见坑点**：120 中自底向上比自顶向下更简洁（不需要额外处理边界）；63 中障碍格子的 dp 值为 0；5 中 DP 的遍历顺序（长度从小到大）；97 中 dp 数组的大小是 (m+1)×(n+1)。  
**自测问题**：能写出 64 网格 DP 的转移方程和初始化吗？97 的 dp[i][j] 代表什么含义？

## Day 32 — day32-dp-2d（2D DP 冲刺）

**核心能力**：编辑距离、最大正方形、股票 K 次交易。  
**当天重点**：72 编辑距离是经典 2D DP；221 最大正方形以右下为顶点的边长；188 股票 IV 是 K 次交易的状态机，可复用 123 思路。  
**拓展**：10 Regular Expression Matching（正则 DP）、44 Wildcard Matching（通配符 DP）

**技能标签**：编辑距离、最大正方形、股票 K 次交易状态机。  
**选题逻辑**：72 编辑距离是二维 DP 的经典中的经典，面试高频；221 最大正方形以右下角为顶点，`dp[i][j] = min(上, 左, 左上) + 1`；188 股票 IV 是 123 的通用化，k 次交易对应 2k 个状态。  
**思维套路**：  
- 编辑距离：`dp[i][j]` = word1 前 i 个字符变成 word2 前 j 个字符的最少操作数；相等则 `dp[i-1][j-1]`，不等则 `min(插入, 删除, 替换) + 1`。  
- 最大正方形：以 (i,j) 为右下角的最大全 1 正方形边长 = `min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) + 1`。  
- K 次交易：`dp[k][0/1]` 表示第 k 次交易、持有/不持有的最大利润；k 很大时退化为无限次交易（122）。  
**题目对应能力**：72 三种操作对应三个方向的转移；221 正方形边长由三个方向的最小值决定；188 K 次交易状态机，复用 123 思路扩展到 k 次。  
**常见坑点**：72 中初始化首行首列（`dp[i][0]=i, dp[0][j]=j`）；221 中矩阵元素是字符 '1' 而非整数 1；188 中 k ≥ n/2 时直接退化为无限次交易避免超时。  
**自测问题**：能写出 72 编辑距离的完整转移方程吗？221 为什么取 min 而不是 max？188 中 k 很大时为什么可以退化为 122？

| ------- | ---------------------------------- | ------ | ---------- |
| 72      | Edit Distance                      | Medium | 经典编辑距离 DP  |
| 221     | Maximal Square                     | Medium | 网格最大正方形 DP |
| **188** | Best Time to Buy and Sell Stock IV | Hard   | K 次交易 DP   |
